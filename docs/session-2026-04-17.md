## Supplier Auto-Detect (Payment Voucher + Official Receipt)
- 3-level supplier resolution from bank transaction description:
  1. **Supplier alias** (DB) ‚Äî `SupplierAlias` table learned from past vouchers/receipts
  2. **Supplier name match** ‚Äî exact/partial match against existing suppliers
  3. **Name extraction** ‚Äî picks name-like line from description (alphabetic, 2+ words, no bank keywords)
- New API endpoints: `GET /api/suppliers/by-alias`, `POST /api/suppliers/alias`
- After creating voucher/receipt, name-like description lines saved as aliases for future auto-match
- Applied to both "Paid To" (voucher) and "Received From" (receipt)
